Basic implementationRealtime validation feedback

This example shows a basic implementation. While the user types, Aragonite runs a sanitation process on the client side to make sure all the fields ar filled correctly.

If the field is invalid it will add a class called ".required" to the HTML element. You customize that class and comunicate to the user that something is wrong with their inputs.

By the time the user finished filling the form, if there is a field with the class required, it will return false preventing the user from submiting wrong data or data-types.

  • javaScript
  • HTML
  • CSS
                        
<script type="text/javascript">
    aragonite.init('demo-form');
</script>


                        
<form id="demo-form" class="how-form" action="#">
        <input type="text" data-type="email" placeholder="Type an Email address" />
    <input type="text" data-type="string" placeholder="Type any string" />
    <input type="text" data-type="phone" placeholder="Type a phone number" />
</form>



.required{
    border:thin solid @light-red !important; 
    background:#FCF0F0 !important;
}

CustomCSS ClassName

By default Aragonite will add a class attribute to your HTML field when something is wrong. This className is call 'required'. However... you might have another CSS class already using that name in your project. If that's the case... you can always overwrite Aragonite's default behavior and pass a custom class name.

If the field is invalid it will add a class called ".customClass" to the HTML element. You customize that class and comunicate to the user that something is wrong with their inputs.

  • javaScript
  • HTML
  • CSS
                        
//className
<script type="text/javascript">
    aragonite.init('demo-form', {
        className: 'myCustomClass'
    });
</script>

    
                        
<form id="demo-form" class="how-form" action="#">
    <input type="text" value="" data-type="email" placeholder="Type an Email address" />
    <input type="text" value="" data-type="string" placeholder="Type any string" />
    <input type="text" value="" data-type="phone" placeholder="Type a phone number" />
</form>
    


    .customClass{
    border: thin solid @light-red !important;
    background: @light-red  !important;
    color:#FFF;
}

CustomValidation

This demo shows a set of options available to validate your forms. I will be adding more options "every week". In case you need to use a custom validation you can always pass it as an argument and then use it on your form.

You pass the property name of these regex through your data-type attribute and Aragonite will do the rest for you. You can pass as many regex as you want*.

  • javaScript
  • HTML
  • CSS
                        
//className
<script type="text/javascript">
    aragonite.init('demo-form', {
        regex: [
                {
                    name: 'customCVV',
                    regex: {
                        value: /^[A-Z][a-z0-9_-]{3,19}$/,
                        message: 'This custom CVV number is wrong'
                    }
                },
                {
                    name: 'customString',
                    regex: {
                        value: /^[a-zA-Z]{7}$/,
                        message: 'Another error'
                    }
                }
        ]
    });
</script>

    
                        
<form id="demo-form" class="how-form" action="#">
    <input type="text" data-type="customCVV" placeholder="Type any string" />
    <input type="text" data-type="customString" placeholder="Type a phone number" />
</form>
    


.required{
    border:thin solid @light-red !important; 
    background:#FCF0F0 !important;
}
